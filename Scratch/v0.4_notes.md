# TorBox Manager — v0.4 Planning Notes

Ideas and observations from the v0.3 session, in rough priority order.

---

## Confirmed bugs / known gaps

**Multi-file torrent downloads always use files[0]**
Worker.py `_request_link` grabs `files[0].id` for all torrent/magnet downloads.
For single-file torrents this is fine. For multi-file sets (like the HyperSpin ROMs)
it silently downloads only the first file. A file picker dialog is needed before
the DownloadWorker is dispatched. The dialog should list `item["files"]` with name
and size for each entry, allow single or multi-select, and submit one DownloadWorker
per selected file.

**Usenet items show hash as name**
Items like `694f6fe710f5796664a96d6f` — TorBox is returning the hash as the `name`
field for some usenet downloads. Nothing we can fix on our end, but worth investigating
whether TorBox populates a different field (e.g. `original_name`, `title`) that would
be more readable. Could add a fallback: if name looks like a bare hex hash, show it in
a dimmed/italic style to signal it's not a real name.

**Pause polling when minimized with no active downloads**
Currently PollWorker fires every N seconds regardless of window state or queue activity.
When minimized and nothing is downloading locally, polling is wasted. Check
`self.isHidden()` and `len(self._downloading) == 0` before submitting PollWorker.
Should still poll on a longer interval (e.g. 5 min) to catch new items added from
other clients.

## Feature ideas

**Multi-file torrent picker dialog**
See above. Priority: high — affects every multi-file torrent download.
Design: modal dialog, table with checkbox per file (name, size columns), Select All /
Deselect All, OK dispatches one DownloadWorker per checked file.

**Download queue / concurrency limit**
Currently all DownloadWorkers run in parallel up to QThreadPool's default limit.
For large sets (Download All on 8 items) this saturates the connection. A simple
queue with a max-concurrent setting (default 2-3) would be more practical.

**Right-click row context menu**
Currently the only row interaction is the Download and Delete buttons. A context menu
on right-click would be a natural place for:
- Copy name to clipboard
- Copy download link to clipboard (resolves and copies the TorBox URL)
- Open in browser (for webdl items)
- Select files (for multi-file torrents — alternative to the picker dialog above)

**Download speed / upload speed columns**
TorBox returns `download_speed` and `upload_speed` (bytes/sec) in the item data.
Confirmed available in v0.2 spec. Could be optional columns alongside Seeds/Peers.
Only meaningful during active TorBox-side download; show "—" otherwise.

**Notification on download complete**
`QSystemTrayIcon.showMessage()` is already available via `self._tray`. When a
DownloadWorker emits `finished`, show a tray notification with the filename.
Could be a toggle in Settings (default on).

**Smarter "Added" column for in-progress items**
Currently "Added" shows `created_at` from TorBox. Could also show `cached_at` for
ready items to indicate when they became available, which is more useful than when
they were originally added.

## Polish / minor

**Log strip auto-clear on size**
`_log_lines` grows unbounded. After ~500 lines trim the oldest 250. The visible
QTextEdit already scrolls, so this only affects the stored list used for filter
re-render and memory.

**Settings: remember window geometry**
Save and restore window position/size on non-maximized launches. Currently always
opens maximized; if the user un-maximizes and moves it, that's lost on restart.
Store `geometry` in config.json via `saveGeometry()` / `restoreGeometry()`.

**About dialog: add build date**
Currently shows version only. Adding the build/release date would help when
diagnosing issues from logs or screenshots.

**Error state: retry button**
When a row shows STATUS_ERROR, the Download button is disabled. A "Retry" button
(or repurposing the Download button to say "Retry" on error) would allow the user
to attempt the download again without having to delete and re-add.

**api.py: user_agent header**
Currently sends no User-Agent header. TorBox's API doesn't require one but it's
good practice to identify the client. Add `"User-Agent": f"TorBoxManager/{APP_VERSION}"`.

## Technical debt

- `worker.py`: DownloadWorker imports `time` and `urllib.parse` inline inside `run()`
  and `_extract_filename()`. Should be top-level imports.
- `ui.py`: `_open_in_explorer` uses `subprocess.Popen` with a string command which
  is fragile on paths with spaces. Should use the list form or `os.startfile`.
- `constants.py`: `COL_VISIBILITY_KEYS` maps column index → config key string.
  If columns are ever reordered the indices need updating in two places. Consider
  a single source of truth (e.g. a list of column descriptor dicts).
